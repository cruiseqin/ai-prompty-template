## 一、需求分析类（15 个）

### 模板 001：PRD 技术评审
```
你是一个有 10 年经验的技术负责人，擅长从产品需求中识别技术风险。

请分析以下 PRD 文档，输出：

1. **功能清单**：按优先级排序（P0/P1/P2）
2. **技术难点**：识别潜在的技术挑战
3. **边界场景**：列出所有可能的异常流程
4. **数据模型**：建议的数据库表结构
5. **接口设计**：RESTful API 初步设计
6. **风险评估**：哪些需求可能延期，为什么

PRD 内容如下：
[粘贴 PRD 文档]
```

## 🎁  bonus：高级用法

### 组合使用技巧

**技巧 1：链式提问**
```
第一轮：帮我生成一个用户登录接口
第二轮：给这个接口加上 JWT 认证
第三轮：加上图形验证码功能
第四轮：优化性能，加上缓存
第五轮：生成对应的单元测试
```

**技巧 2：角色扮演**
```
你现在是一个有 15 年经验的技术专家，请用通俗易懂的方式给我解释 [复杂概念]。
```

**技巧 3：对比分析**
```
请对比方案 A 和方案 B 的优缺点，并给出在什么场景下选择哪个方案。
```

**技巧 4：代码重构**
```
这段代码有哪些可以优化的地方？请给出重构后的版本，并说明优化理由。
```

**技巧 5：渐进式完善**
```
这是初版设计，请指出问题 → 根据反馈修改 → 再次评审 → 最终定稿
```

1. **不要照搬**：AI 生成的代码要理解后再使用

4. **积累沉淀**：把好用的提示词加入个人知识库
5. **分享交流**：和团队分享高效用法
